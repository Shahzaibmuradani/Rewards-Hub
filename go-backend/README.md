# Rewards Hub Backend

Go backend for the Rewards Hub mobile app. It exposes reward, goal, and fastest-path APIs backed by PostgreSQL and seeds a demo user on startup.

## Layout

```text
go-backend/
├── cmd/
├── internal/
├── migrations/
├── .env.example
├── go.mod
├── go.sum
├── Makefile
└── README.md
```

## Requirements

- Go 1.25 or newer
- PostgreSQL 12 or newer
- Optional: `GEMINI_API_KEY` for personalized recommendation text

## Setup

1. Create the database and user.

```bash
createdb rewards_hub
```

2. Copy the example env file.

```bash
cp .env.example .env
```

3. Configure `.env`.

```env
PORT=8080
DB_HOST=localhost
DB_PORT=5432
DB_USER=postgres
DB_PASSWORD=postgres
DB_NAME=rewards_hub
GEMINI_API_KEY=
GEMINI_MODEL=gemini-2.5-flash
```

4. Install Go dependencies.

```bash
go mod download
go mod tidy
```

## Run

```bash
go run ./cmd/main.go
```

Or use the Makefile:

```bash
make run
```

If `go` is not on your PATH on Windows, use the bundled launcher:

```powershell
powershell -ExecutionPolicy Bypass -File .\run-backend.ps1
```

Or from `cmd.exe`:

```bat
run-backend.cmd
```

The server listens on `http://localhost:8080` unless `PORT` is set.

## API

### `GET /api/user-goals?user_id=...`
Returns the user profile, wallet summary, and the active goal list.

### `GET /api/goals`
Returns the catalog of selectable rewards.

### `GET /api/fastest-path?user_id=...`
Returns ranked games that help the user reach the goal fastest.

### `POST /api/user/select-goal?user_id=...`
Updates the active goal.

Request body:

```json
{
  "catalog_id": "6f1b6f79-7b93-4b4d-933d-0d7c01c5a5c1"
}
```

## Notes

- Progress calculations are done server-side on each request.
- Recommendation text is generated by the AI service when `GEMINI_API_KEY` is available.
- The backend creates tables automatically at startup.

## Troubleshooting

- `User not found`: confirm the demo user exists or send a valid `user_id`.
- Connection errors: verify PostgreSQL is running and the `.env` credentials are correct.
- Port conflicts: change `PORT` before starting the server.
